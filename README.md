# Shape Palette — PowerPoint 図形パレット アドイン

図形をタブ別に並べておき、クリック一発でスライドに挿入する Office.js タスクペインアドイン。
挿入後は PowerPoint 上で普通にドラッグして配置する想定。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `taskpane.html` | パネル本体（UI + 取り込み/挿入ロジック、単一ファイル完結） |
| `commands.html` / `commands.js` | ショートカットから呼ばれる関数（表示/非表示） |
| `shortcuts.json` | キー定義（`Ctrl+Shift+1` 表示 / `Ctrl+Shift+2` 非表示） |
| `manifest.xml` | アドインのマニフェスト |

## できること

- **タブ**: お気に入り / 図形 / 矢印 を初期搭載。右上の鉛筆（編集モード）で
  タブ名の変更・追加(＋)・削除ができる。タブ内の項目も移動・削除・ジオメトリ変更が可能。
- **クリックで挿入**: タイルをクリックすると現在のスライド中央付近に図形を挿入。
- **取り込み**: スライド上で図形を選択 →「選択中の図形を取り込む」で、サイズ・塗り・線・
  テキストを取り込む。※ Office.js では既存図形の「ジオメトリ種別」が読めないため、
  取り込み後は角丸四角になる。編集モードのドロップダウンで正しい形に変更できる。
- **お気に入り**: 各タイルの★でお気に入りタブへ。
- **書き出し/読み込み**: 右上「…」から JSON でエクスポート/インポート。PC間の持ち運び用。
- **永続化**: `OfficeRuntime.storage` に保存（PowerPoint を閉じても残る）。

> ブラウザで `taskpane.html` を直接開くと **demo モード**で動作（挿入はトースト表示のみ）。
> UI と操作感の確認用。

## セットアップ（ローカル開発）

前提: Node.js（最新 LTS）。

```bash
# 1) 配信用の簡易HTTPSサーバを用意（例: office-addin-dev-certs + http-server）
npm install -g office-addin-dev-certs http-server
npx office-addin-dev-certs install        # localhost 用の信頼済み証明書

# 2) このフォルダを https://localhost:3000 で配信
http-server . -S \
  -C "$(npx office-addin-dev-certs install --machine 2>/dev/null; echo ~/.office-addin-dev-certs/localhost.crt)" \
  -K ~/.office-addin-dev-certs/localhost.key \
  -p 3000
```

> アイコン参照（`assets/icon-*.png`）はマニフェストに含めてあるので、`assets/` に
> 16/32/80px の PNG を置くか、参照行を一旦削っても動作確認は可能。

### サイドロード（手動）

`manifest.xml` の `<Id>` を一意の GUID に置き換えてから読み込む。

- **Windows**: 共有フォルダにマニフェストを置き、PowerPoint の
  「ホーム → アドイン → その他のアドイン → 個人用アドイン → 共有フォルダ」から追加。
- **Mac**: `~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/` に
  `manifest.xml` を置いて PowerPoint を再起動。
- **Web**: PowerPoint on the web の「アドイン → 個人用アドイン → マイ アドインのアップロード」。

> `yo office`（Yeoman generator for Office Add-ins）でプロジェクト雛形を作り、
> 生成された `webpack` 配信に上記4ファイルを差し込む方法でも可。その場合は
> `npm start` がサイドロードまで自動でやってくれる。

## 使い方

1. `Ctrl+Shift+1`（Mac は `Cmd+Shift+1`）でパネルを開く。
2. **置きたい場所の近くの図形を選択**してから、タイルをクリック → その図形の右隣に挿入される
   （何も選択していなければスライド中央付近に挿入）。挿入したものは「最近」タブに自動でたまる。
3. キーボードだけでも操作できる: `1`〜`9` で番号挿入、`↑↓←→`＋`Enter`、`[` `]` でタブ切替。
4. 挿入された図形を PowerPoint 上でドラッグして微調整。
5. よく使う見た目（塗り・枠・テキスト入り）はスライドで作って「取り込む」。
   **複数選択して取り込むと複合パーツ**になり、配置ごと再利用できる。

## 主な機能

- **タブ**: お気に入り★ / 最近🕒（自動）/ 図形 / 矢印。鉛筆ボタンで通常タブの名前変更・追加・削除、
  タブ内項目の移動・削除・形変更ができる（お気に入り・最近は固定タブ）。
- **クリック挿入**: タイルを押すと、選択中の図形の右隣（なければスライド中央）に挿入。
- **キーボード挿入**: 数字 `1`〜`9` で番号挿入、`↑↓←→` で選択移動＋`Enter`、`[` `]` でタブ切替、`Esc` で解除。
- **最近（履歴）**: 挿入したものが自動で「最近」タブ先頭に積まれる（最大24件・重複は先頭へ）。
  メニューから履歴クリア可。
- **複合パーツ**: 複数図形を選択して取り込むと「まとめて1つ／個別に分ける」を選択可。
  まとめた場合は相対配置ごと保存し、挿入時にその配置で一括展開する。
- **お気に入り / JSON書き出し・読み込み**: 持ち運び・共有用。

## 取り込みで保存される内容

- **塗り色**（`fill.foregroundColor`）／**枠の色・太さ・線種**（`lineFormat.color/weight/dashStyle`）
- **テキスト**と**フォントの色・サイズ・太字**
- サイズ（幅・高さ）

→ 「青枠・水色背景の角丸四角」「色違い」「テキスト入りの同じ角丸四角」などは見た目そのまま再現される。

## 既知の制約（Office.js 由来）

- **テーマの色で塗った図形は塗り色が黒(#000000)として読まれる**（Office.js のバグ）。
  登録用の図形は「塗りつぶしの色 → 標準の色 / ユーザー設定(RGB)」で塗っておくこと。
  取り込み時に黒が来たら警告を表示する。
- 既存図形の **ジオメトリ種別は読み取れない** → 取り込みは角丸四角になる。
  別の形にしたい場合は編集モードのドロップダウンで変更（用途が角丸四角中心なら気にならない）。
- **グラデーション塗り・グループ図形・アニメーションの再現は不可**。
  単色塗りの図形・矢印・テキスト入りパーツ向けの設計。
- **「画面表示の中央」への挿入は不可**（PowerPoint の Office.js に表示領域取得 API がないため）。
  代替として「選択図形の隣」方式を採用。真のドラッグ&ドロップ挿入も非対応。
